# תפקידי צוות האיכות

## Orchestrator

אחריות:

- מגדיר את היקף הסבב.
- בוחר אילו דרישות נבדקות.
- מריץ או מבקש להריץ `npm run qa`.
- מאחד ממצאים כפולים.
- מתעדף תיקונים לפי severity וסיכון.

שאלות חובה:

- איזו התנהגות אמורה להשתנות?
- איזו התנהגות חייבת להישאר זהה?
- האם כשל אוטומטי חוסם פריסה?
- האם יש החלטה כתובה בסוף הסבב?

## Product QA

אחריות:

- משווה את האפליקציה ל-`requirements.md`.
- מסמן דרישות חסרות או לא מדויקות.
- בודק flows מרכזיים עם נתוני דוגמה.

תוצרים:

- טבלת קבלה מעודכנת.
- ממצאים עם Requirement ID.
- הצעה לעדכון דרישות כאשר התנהגות רצויה אינה מתועדת.

## UX/A11y Reviewer

אחריות:

- בודק קריאות, עומס, מצבי empty/loading/error ו-mobile.
- בודק התאמה לעברית ו-RTL.
- בודק נגישות בסיסית במקלדת ובשמות נגישים.

תוצרים:

- ממצאים עם severity.
- המלצות קצרות שאפשר לתקן בפועל.
- ציון אם הבעיה חוסמת שימוש או רק ליטוש.

## Engineering Reviewer

אחריות:

- בודק איכות קוד, טיפוסים, hooks ו-Supabase.
- מוסיף או מעדכן בדיקות.
- מתקן באגים שסוכמו בדוח.

תוצרים:

- patch ממוקד.
- בדיקות רגרסיה.
- דוח קצר של סיכון שנותר.
