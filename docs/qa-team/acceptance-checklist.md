# צ'קליסט קבלה

השתמשו בצ'קליסט הזה בכל סבב בדיקה. סמנו כל סעיף באחד המצבים: `Pass`, `Partial`, `Fail`, `Blocked`, `Not Tested`.

## לפני הבדיקה

| סעיף | סטטוס | הערות |
| --- | --- | --- |
| הדרישות הרלוונטיות מסומנות ב-`requirements.md`. |  |  |
| סביבת `.env.local` קיימת ואינה מכילה ערכים אמיתיים בדוח. |  |  |
| יש נתוני בדיקה לפרויקטים, משימות, משימות ללא פרויקט וקבצים. |  |  |
| ברור אילו מסכים או קומפוננטות השתנו. |  |  |

## מוצר

| דרישה | מה לבדוק | סטטוס | ממצא |
| --- | --- | --- | --- |
| AUTH-001 | משתמש לא מחובר לא רואה נתוני פרויקטים. |  |  |
| AUTH-003 | התנתקות סוגרת את הסשן ומחזירה למסך כניסה. |  |  |
| SCP-001 | עבודה ואישי מציגים נתונים נפרדים. |  |  |
| SCP-002 | בחירת scope נשמרת אחרי רענון. |  |  |
| PRJ-001 | יצירה, עריכה והקפאה/מחיקה של פרויקט. |  |  |
| PRJ-003 | כל אפשרויות המיון מחזירות סדר צפוי. |  |  |
| PRJ-005 | בחירת פרויקט מסננת משימות לפי הפרויקט הנבחר. |  |  |
| TSK-001 | יצירה, עריכה, השלמה והקפאה/מחיקה של משימה. |  |  |
| TSK-002 | משימה ללא פרויקט מופיעה בתצוגת "ללא פרויקט". |  |  |
| TSK-004 | checkbox שומר השלמה מיידית ומעדכן `closed_at`. |  |  |
| TSK-005 | הסתרת משימות שהושלמו עובדת רק בתצוגות פעילות. |  |  |
| FILE-001 | ספירת קבצים מוצגת על כרטיס הפרויקט. |  |  |
